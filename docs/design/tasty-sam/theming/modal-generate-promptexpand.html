⚠ 与现状/我先前假设的差异 & 交互风险（落地前必读）

这两个模态曾被我先前粗略归为「批量参数」「双语展开」，实际并非如此。以下为关键纠偏，风险等级偏高，请重点 Review。

StoryboardGenerateDialog
维度现状（真实组件）本补稿处理风险
本质纠偏它是「前置检查 + 覆盖警告 + 确认」弹窗，不暴露 范围/数量/画风/模型 任何参数；真正生成在父级 toast 流跑不臆造参数控件；只还原两项检查 + 警告 + 确认高·纠偏
两项检查剧本 ≥40 字 / 已抽取实体（>0）。pass 绿、fail amber，fail 带 hint + 「去 Script 步骤补充」展示双 pass；表下说明 fail 态与首次态一致
覆盖警告allPass 且已有分镜>0 才出 amber「清空现有 N 个分镜…无法撤销」展示 N=8 覆盖警告；Confirm 文案「重新生成」低
确认即关闭handleConfirm 先关弹窗再 onConfirm()，弹窗内无 loading，进度走全局 toast + GenerationBanner无内置 loading 态；落地须保留此时序需确认
tokenbackdrop 用 bg-overlay，panel bg-elevated，footer 用 WorkflowActionButtonbackdrop→--color-overlay，panel→--color-bg-elevated，按钮还原 ghost/pri一致
PromptExpandModal
维度现状（真实组件）本补稿处理风险
双语纠偏本模态单语——只有 1 个 textarea，无语言切换；「双语 CN/EN + Copy/Apply + 再优化」是同级 PolishPanel 的能力只画单 textarea + 字符计数；表注明双语归属 PolishPanel高·纠偏
快捷键Esc 关闭丢弃草稿；Cmd/Ctrl+Enter 与 Cmd/Ctrl+E 均保存；入口图标 + 文本框 Cmd/Ctrl+E 打开页脚双提示 1:1；入口说明在表下一致

草稿丢弃本地 draft；Cancel/Esc 丢弃，Save 提交 onSave(next) 回写父级静态展示；落地须保留本地草稿语义低
